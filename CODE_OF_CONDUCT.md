# Code of Conduct

Short version: **be cool to other people. Don't be a jerk.**

If you can read that and apply it without needing a checklist, you're done. Go contribute.

The rest of this file is for the edge cases.

## The actual rules

- **Be respectful.** Disagree about code all you want — never about identity.
- **Assume good faith.** Most "bad" PRs are someone trying to help while still learning the codebase. Help them get unstuck instead of dunking on them.
- **No harassment, period.** Personal attacks, slurs, sexual attention, doxxing, sustained hostility — out, regardless of how clever or "ironic" you think you're being.
- **Public criticism, private corrections.** If someone's code has a bug, comment publicly — that's the point of review. If someone's *behavior* has a problem, message a maintainer privately first. Don't pile on in threads.
- **English in public spaces.** Use whatever language you want in your fork or your DMs. Here, English keeps the door open for everyone who shows up.
- **Don't make this place exhausting.** Sealioning, derailing, "well, technically", relentless contrarianism — all corrosive even when individually polite. Read the room.

## Scope

Wherever this project lives: issues, PRs, discussions, any Discord/forum if one ever exists, conferences where StinkOS comes up, your fork's README if it points back here.

## What happens if you break the rules

1. **First time:** a maintainer messages you. You hear what landed wrong, you adjust, life goes on. Most things end here.
2. **Pattern:** public note from a maintainer + temporary timeout from the repo.
3. **Bad-faith pattern:** banned. New accounts created to evade the ban → also banned, on sight.

I (jovinull) am the maintainer right now, which means I'm also the enforcement. If StinkOS ever has multiple maintainers and your complaint is about one of them — including me — reach the others directly.

## Reporting

Email **felipejovinogamerplay@gmail.com**, or open an issue and ping `@jovinull` for stuff already public and clearly not severe.

You don't need to be the target to report something. If you see behavior that's making other people unwelcome, say so — even quietly, in a DM. That's how communities stay livable.

## A note on intent

"I didn't mean it that way" is not a defense, but it is information. Most rough interactions in open source come from text-only communication, language barriers, and people being tired. The point of this document isn't to punish — it's to give us a way to fix things when they go sideways, and to make sure they don't keep going sideways with the same person, again and again.

Be cool. Build neat stuff. That's the whole game.
